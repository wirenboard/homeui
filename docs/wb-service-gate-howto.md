# Закрыть свой сервис авторизацией Wiren Board (порт-гейт)

Сервис живёт на отдельном **порту**; nginx пускает на него только залогиненных в
homeui с нужной ролью: браузер перекидывает на форму входа, API-клиент получает
`401`. Требует homeui с gate-сниппетами и лимитом `/auth/check` 1000/мин — на более
старом сначала обновить homeui.

Механика: сервис слушает **только `127.0.0.1`** (снаружи невидим), nginx на внешнем
порту делает `auth_request` к `/auth/check`. Внешний порт = **«2» перед внутренним**
(`1880 → 21880`, `9000 → 29000`); совпадать они не могут — nginx на `0.0.0.0:<порт>`
перекрыл бы сам сервис.

## Настройка

### 1. Сервис на loopback

Сервис должен слушать `127.0.0.1:<внутренний-порт>`, не `0.0.0.0` — иначе внутренний
порт обходит гейт.

### 2. Файл гейта

Убедиться, что внешний порт свободен
(`ss -ltn | awk '{print $4}' | grep -E '[:.]29000$'` — пусто = свободен), затем
`/etc/nginx/conf.d/<сервис>.conf` — поменять 4 отмеченные строки:

```nginx
server {
    listen 29000 ssl;                              # (1) внешний порт (= 2 + внутренний)
    server_name _;
    include /etc/nginx/snippets/wb-gate-tls.inc;

    location / {
        set $wb_role   admin;                      # (2) минимальная роль: admin | operator | user
        set $wb_return /open-my-service;           # (3) путь возврата (см. шаг 3)
        auth_request /wb_auth_check;
        error_page 401 = @wb_unauth;
        proxy_pass http://127.0.0.1:9000;          # (4) внутренний адрес сервиса
        include /etc/nginx/snippets/wb-gate-proxy.inc;
    }
    include /etc/nginx/snippets/wb-gate-authcheck.inc;
    include /etc/nginx/snippets/wb-gate-unauth.inc;
}
```

Роль работает **«эта и выше»** (`user < operator < admin`): `user` пускает любого
залогиненного — выбирать осознанно.

Гейт должен быть в том же режиме, что и homeui (Настройки → Веб-интерфейс):

- **HTTPS включён** — конфиг как выше; гейт использует сертификат homeui.
  Пока в настройках нет статуса «сертификат получен», гейт отдаёт
  сертификат-заглушку и браузер будет предупреждать.
- **HTTPS выключен** — уберите `ssl` из `listen` и строку с `wb-gate-tls.inc`,
  в шаге 3 у bounce поставьте `http`. Сертификат не нужен.

Переключили HTTPS в настройках — переведите гейт в соответствующий вариант.

### 3. Редирект под путём (рекомендуется)

`/etc/nginx/includes/default.wb.d/<сервис>-bounce.conf` — порт как в `listen`,
схема (`https`/`http`) как у варианта гейта из шага 2:

```nginx
location = /open-my-service { return 302 https://$host:29000/; }
```

После логина homeui возвращает только на **свой origin** (другой порт отбрасывается
как open-redirect), поэтому `$wb_return` указывает на этот путь, а он уже
302-редиректит на порт. Без файла гейт работает, но после логина пользователь
окажется на дашборде homeui.

### 4. Применить

```bash
nginx -t && systemctl reload nginx
```

Если `nginx -t` ругается — исправить файл и **не** перезагружать: битый конфиг
уронит следующий reload вместе с веб-интерфейсом.

### 5. Пункт в меню «Интеграции» (опционально)

`/usr/share/wb-mqtt-homeui/custom-menu/<сервис>.json` — применяется без рестарта,
`url` — путь из шага 3; обновить страницу homeui — пункт появится:

```json
{
  "id": "integrations",
  "children": [
    {
      "id": "my-service",
      "url": "/open-my-service",
      "isExternal": true,
      "openInNewTab": true,
      "title": { "ru": "Мой сервис", "en": "My Service" }
    }
  ]
}
```

## Как пользоваться

- Прямой вход: `https://<хост>:29000/`; удобный: `https://<хост>/open-my-service`
  (или пункт меню) → редирект на порт.
- Открывать по **тому же hostname, что и homeui** (cloud-DNS
  `<ip>.<serial>.ip.wirenboard.com`) — сессионная кука host-only, на другом имени
  пойдёт бесконечный редирект.

## Ограничения

- **Облачный тоннель Wiren Board** пробрасывает только основной веб-порт (443) —
  порт-гейт работает в локальной сети / при прямом доступе.
- **API-клиенты (скрипты)**: логин `POST /auth/login`, дальше кука `id` с каждым
  запросом; без куки — `401`.
- **Выключение HTTPS в homeui не гасит TLS-гейт, но останавливает продление
  сертификата** — со временем браузеры начнут блокировать гейт; выключили HTTPS
  насовсем — перевести гейт на HTTP-вариант (шаг 2).
