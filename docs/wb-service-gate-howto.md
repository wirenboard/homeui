# Закрыть свой сервис авторизацией Wiren Board (порт-гейт)

Сервис живёт на отдельном порту; nginx пускает на него только залогиненных в homeui
с нужной ролью: браузер перекидывает на форму входа и возвращает обратно, API-клиент
получает `401`. Гейт **сам следует режиму HTTPS** homeui: переключили тумблер в
настройках — все гейты автоматически перенастроились на TLS или обычный HTTP.

## Настройка

1. Сервис должен слушать **только `127.0.0.1:<порт>`** — иначе внутренний порт
   обходит гейт.

2. Положить `<имя>.json` (имя — строчная латиница/цифры/дефис) в каталог гейтов:
   - `/usr/share/wb-mqtt-homeui/gates.d/` — для гейтов, которые ставятся **пакетом**
     (файл живёт и обновляется вместе с пакетом);
   - `/etc/wb-homeui/gates.d/` — для гейтов, которые админ настраивает **руками**
     (переживают обновление прошивки). Файл с тем же именем в `/etc` полностью
     заменяет пакетный — так можно переопределить гейт пакета.

   ```json
   {
     "internalPort": 9000,
     "externalPort": 29000,
     "role": "admin",
     "menu": { "title": { "ru": "Мой сервис", "en": "My Service" } }
   }
   ```

   | Поле | Обязательное | Что делает |
   |---|---|---|
   | `externalPort` | да | внешний порт гейта (1024–65535); не должен совпадать с портами других гейтов и их сервисов |
   | `internalPort` | да | порт сервиса на `127.0.0.1` (тоже 1024–65535); не должен совпадать с внешним |
   | `role` | нет (`admin`) | минимальная роль, «эта и выше»: `user < operator < admin`; `user` пускает любого залогиненного |
   | `auth` | нет (`true`) | `false` — пустить **без авторизации** (доверенные `X-WB-*` всё равно зачищаются) |
   | `menu` | нет | пункт в меню «Интеграции»; `menu.title` — подписи `ru`/`en`, достаточно одного языка. У ролей ниже `role` пункт скрыт — но это только видимость, доступ контролирует сам гейт |

3. Проверить и применить:

   ```bash
   wb-homeui-gates check   # показать, какие гейты получатся; битые файлы — с причинами
   wb-homeui-gates apply   # создать конфиги nginx, проверить (nginx -t) и перезагрузить nginx
   ```

   Если nginx отверг новую конфигурацию, `apply` печатает причину и возвращает
   nginx к прежнему рабочему состоянию — сломать nginx или соседние гейты нельзя.
   Откатываются только сгенерированные конфиги: ваши файлы в `gates.d` остаются как есть —
   поправьте ошибку и запустите `apply` снова. Backend сам переприменяет гейты
   при старте и при переключении тумблера HTTPS.

   Код выхода: `0` — все конфиги валидны и применены; `1` — хотя бы один конфиг
   пропущен (остальные `apply` всё равно применил) или применение откатилось.

## Как пользоваться

- Через меню или `<хост>/open-<имя>` — редирект на порт гейта.
- Открывать по **тому же hostname, что и homeui** — сессионная кука host-only,
  на другом имени будет бесконечный редирект на логин.

## Как это устроено

- Готовые конфиги nginx лежат в `/var/lib/wb-homeui/nginx-gates/` — они создаются
  автоматически, редактировать их руками не нужно. TLS у гейтов есть, пока тумблер
  HTTPS включён и есть пригодный сертификат, — вместе с основным UI.
- `/etc/wb-homeui` (`gates.d` и `custom-menu`) сохраняется при обновлении прошивки
  через wb-configs — симлинк в `/mnt/data` появляется после перезагрузки.
- Нужны нестандартные nginx-директивы (таймауты, свои location)? Положите файл
  `<имя>.nginx.inc` в тот же каталог, где лежит `<имя>.json`, — его содержимое
  попадёт в server-блок этого гейта. Ошибку в нём поймает `nginx -t` при `apply`:
  применение откатится, сам файл никто не удалит — поправьте его и повторите
  `apply`.

## Ограничения

- **Облачный тоннель Wiren Board** пробрасывает только основной веб-порт —
  порт-гейт работает в локальной сети / при прямом доступе.
- **API-клиенты (скрипты)**: логин `POST /auth/login`, дальше кука `id` с каждым
  запросом; без куки — `401`.
- Занятость внешнего порта другим приложением `nginx -t` не ловит — если гейт не
  отвечает после `apply`, проверьте `ss -ltn` и журнал nginx.
- Запросы к гейту лимитируются по IP клиента (15 r/s, всплеск до 200) — устойчивое
  превышение даёт `429`. Обычному UI за гейтом этого хватает с запасом.
